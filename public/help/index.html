About

Created by Remy Sharp as a child of the late 70s the ZX Spectrum are his fondest memories of computing as a kid. So it only made sense that he turn his browser-based skills to made things work on the Spectrum.

Inspired by the return of Spectrum via the ZX Next, Remy learnt how loading screens worked, how the binary data in screens is arranged and presented his work in a mad demo on stage in 2018.

Open Source

The code is a little hacky (written over a few days), but has some semblance of organisation. It can be found in github and is available under the MIT license:

https://github.com/remy/zx-tools

Text 2 BASIC

Features:

  • Drag and drop support
  • Can upload and parse 3DOS+ .bin files and plain .txt
  • Export as:
    • .TAP
    • .bin 3DOS+ format - compatible with ZX Spectrum Next
    • .wav with correct headers and pilot for tape loading
  • Import from gists via query string, such as:
    ?url=https://gist.github.com/remy/3cbe71821d3eae96273d0cef0426fda0 or
    ?url=3cbe71821d3eae96273d0cef0426fda0
  • Parses 10print"ok" and correctly expands
  • Unique line numbers, so repeating a line number will replace previous line
  • Session remembers last code

Sprites

Editor based on the .spredit tool on the ZX Spectrum Next.

Features:

  • Drag and drop support
  • Exports to .spr ZX Spectrum Next compatible format
  • Imports PNG file format
  • Import support dimensions at multiples of 16x16 blocks - i.e. 32x48 or 16x256 etc
  • Click to paint current colour, shift+click to delete/paint transparency
  • Alt+click on pixel to add the colour to current colours
  • Keys 1-6 to change colour from colour history

SCR tools

.SCR being the exported format of Spectrum screen memory. These tools can render original .SCR files into PNG files.

The tools can also take any JPG image (in any resolution) and convert it to a ZX Spectrum compatible image that be rendered on the Spectrum.
